# Project Structure Guide

This document explains every folder and file in the project in simple terms.

---

## Root Files

| File | What It Does |
|------|-------------|
| `requirements.txt` | Lists all Python packages needed to run the project (PyTorch, OpenCV, scikit-learn, etc.) |
| `setup.py` | Makes the project installable as a Python package (`pip install -e .`) |
| `Makefile` | Shortcut commands to run training, evaluation, EDA, etc. (e.g., `make train`) |
| `README.md` | Project overview and instructions for getting started |
| `.gitignore` | Tells Git which files to ignore (data, checkpoints, virtual env, etc.) |
| `LICENSE` | MIT license for the project |

---

## `configs/` — Configuration Files

| File | What It Does |
|------|-------------|
| `default.yaml` | Main configuration file for the entire project. Controls: learning rate, batch size, number of epochs, data paths, model architecture settings, augmentation, NMS parameters, etc. |
| `custom_anchors.yaml` | Anchor box configurations generated by k-means clustering on SKU-110K bounding boxes. Defines optimal anchor aspect ratios for the dataset. |

---

## `src/` — Source Code (Core Library)

This is the main codebase, organized into subpackages:

### `src/models/` — Neural Network Architecture

| File | What It Does |
|------|-------------|
| `backbone.py` | MobileNetV3-Large backbone — extracts visual features from images at 3 scales (stride 8, 16, 32). Uses depthwise separable convolutions for efficiency. |
| `fpn.py` | Feature Pyramid Network — takes the 3 backbone feature maps and combines them into 5 multi-scale feature maps (P3-P7) for detecting objects at different sizes. |
| `protonet.py` | Prototype Network — generates k prototype masks (image-level mask templates) that are shared across all detections. |
| `prediction_head.py` | Prediction Head — attached to each FPN level, predicts: (1) class scores, (2) bounding box offsets, (3) mask coefficients for each anchor. |
| `detection.py` | Detection module — post-processes raw predictions: decodes boxes, applies confidence thresholding, runs Soft-NMS, and assembles final instance masks. |
| `yolact.py` | Main YOLACT model — ties together backbone + FPN + ProtoNet + prediction heads into a single end-to-end model. |
| `__init__.py` | Package init, exports the YOLACT class. |

### `src/data/` — Data Loading and Preprocessing

| File | What It Does |
|------|-------------|
| `dataset.py` | SKU-110K dataset loader — reads CSV annotations, loads images, applies transforms, handles train/val/test splits. Returns image tensors and bounding box targets. |
| `augmentations.py` | Data augmentation pipeline — random horizontal flip, photometric distortion (brightness/contrast/saturation/hue), random crop with IoU constraint, resize to 550x550, ImageNet normalization. |
| `anchors.py` | Anchor box generator — creates multi-scale anchor boxes at each FPN level with the aspect ratios from k-means analysis. |
| `__init__.py` | Package init. |

### `src/training/` — Training Infrastructure

| File | What It Does |
|------|-------------|
| `trainer.py` | Training loop — handles epoch iteration, forward/backward passes, optimizer stepping, learning rate scheduling (cosine annealing with warmup), checkpointing, and logging. |
| `losses.py` | Loss functions — implements: (1) Focal Loss for classification, (2) Smooth L1 for bounding box regression, (3) Binary Cross-Entropy for mask prediction. Combines them with weights [1.0, 1.5, 6.125]. |
| `__init__.py` | Package init. |

### `src/evaluation/` — Model Evaluation

| File | What It Does |
|------|-------------|
| `evaluator.py` | Runs inference on validation/test set, collects predictions, and computes COCO-style metrics. Generates visualization outputs (detection samples, PR curves, density analysis plots). |
| `metrics.py` | COCO-style mAP computation — calculates AP at different IoU thresholds (0.50, 0.75, 0.50:0.95) and AR at different detection counts (1, 10, 100). Pure NumPy implementation (no pycocotools dependency). |
| `__init__.py` | Package init. |

### `src/deployment/` — Model Export and Optimization

| File | What It Does |
|------|-------------|
| `export_onnx.py` | Converts the PyTorch model to ONNX format for production deployment. Uses opset 13 with dynamic batch axes. |
| `quantize.py` | INT8 post-training quantization via ONNX Runtime — reduces model size and enables integer-only inference on edge hardware. |
| `benchmark.py` | Benchmarks inference speed (latency, FPS) for PyTorch, ONNX FP32, and ONNX INT8 variants. Runs 50 inference passes and reports averages. |
| `__init__.py` | Package init. |

### `src/baseline/` — Classical ML Baseline

| File | What It Does |
|------|-------------|
| `hog_svm.py` | HOG + Linear SVM baseline detector — extracts Histogram of Oriented Gradients features, trains a linear SVM classifier with hard-negative mining, runs sliding-window detection with multi-scale image pyramids. |
| `__init__.py` | Package init. |

### `src/utils/` — Utility Functions

| File | What It Does |
|------|-------------|
| `soft_nms.py` | Gaussian Soft-NMS implementation — instead of hard-thresholding overlapping detections, smoothly decays confidence scores based on IoU overlap. Critical for dense scenes where objects genuinely overlap. |
| `visualization.py` | Visualization helpers — draws bounding boxes and masks on images, creates comparison plots, generates training curve plots. |
| `helpers.py` | General utilities — configuration loading, device setup, logging, seed setting, etc. |
| `__init__.py` | Package init. |

## `scripts/` — Executable Scripts

| File | What It Does |
|------|-------------|
| `train.py` | Main training entry point. Run: `python scripts/train.py --config configs/default.yaml` |
| `evaluate.py` | Runs evaluation on validation set with a trained checkpoint. Run: `python scripts/evaluate.py --checkpoint results/training/checkpoints/best_model.pth` |
| `run_eda.py` | Generates all EDA visualizations (histograms, scatter plots, anchor analysis). Run: `python scripts/run_eda.py` |
| `run_baseline.py` | Trains and evaluates the HOG+SVM baseline. Run: `python scripts/run_baseline.py` |
| `export.py` | Exports model to ONNX and runs deployment benchmarks. Run: `python scripts/export.py --checkpoint results/training/checkpoints/best_model.pth` |
| `download_data.sh` | Shell script to download the SKU-110K dataset from AWS S3. |

---

## `notebooks/` — Jupyter Notebooks (Interactive Walkthroughs)

| Notebook | What It Covers |
|----------|---------------|
| `01_EDA_and_Data_Analysis.ipynb` | Exploratory Data Analysis — loads the dataset, computes statistics (object counts, box sizes, aspect ratios), generates visualizations, runs k-means anchor analysis. |
| `02_Classic_ML_Baseline.ipynb` | HOG+SVM Baseline — demonstrates the classical approach, shows HOG feature visualizations, trains the SVM, evaluates detection performance. |
| `03_DL_Training_and_Evaluation.ipynb` | Deep Learning — walks through model architecture, training configuration, loss curves, and evaluation metrics with the YOLACT model. |
| `04_ONNX_Deployment.ipynb` | Deployment — demonstrates ONNX export, INT8 quantization, and latency benchmarking. |

|------|-----------------|
| `training_log.json` | Complete training history — per-epoch losses (cls, box, mask, total), learning rates, validation losses |

| `detection_samples.png` | Visual examples of model detections on validation images |
| `precision_recall.png` | Precision-Recall curves at different IoU thresholds |
| `density_analysis.png` | How performance varies with object density |

### `results/baseline/` — HOG+SVM Baseline Results
| File | What It Contains |
|------|-----------------|
| `baseline_metrics.json` | Precision, recall, mAP for the HOG+SVM detector |
| `baseline_detections.png` | Visual examples of HOG+SVM detections |
| `hog_features_visualization.png` | Visualization of HOG feature extraction |

### `results/deployment/` — ONNX Export and Benchmarks
| File | What It Contains |
|------|-----------------|
| `yolact.onnx` + `yolact.onnx.data` | ONNX FP32 model (0.5 MB graph + 38 MB weights) |
